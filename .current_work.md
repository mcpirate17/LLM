# Active handoff — nano BLiMP / nano HellaSwag probe design

**Session:** 2026-05-02 ET  •  **Agents:** Tim + Codex, coordinating with Claude

## User goal

Tim looked at real HellaSwag/BLiMP examples and correctly called out that nano
models are not trained on the world knowledge or linguistic breadth those
benchmarks require. The proposed replacement is a nano-scale benchmark:

- very small vocabulary, roughly 20-40 active tokens;
- limited nouns, verbs, adjectives;
- train the same nano model on simple noun→verb and noun→adjective associations;
- evaluate whether it can predict the right association among distractors.

This should answer: "can the architecture learn tiny controlled semantic-style
associations at all?" before we ask it to do real HellaSwag or BLiMP.

## Codex current patch

Codex has started a standalone probe at:

- `research/eval/synthetic_association_eval.py`
- `research/tests/test_synthetic_association_eval.py`

Current shape:

- active vocab default: 40 active token IDs;
- content classes: noun / verb / adjective;
- deterministic mappings:
  - noun -> associated verb
  - noun -> associated adjective
- train a deepcopy of the candidate model on the association mappings;
- evaluate four-way forced-choice accuracy for verbs and adjectives;
- return `synthetic_association_*` fields plus metric version
  `synthetic_association_v1`.

This is intentionally separate from `hellaswag_eval.py`; real HellaSwag
provenance remains clean. If this works, the next step is wiring this as a
candidate nano-understanding metric, not pretending it is the original
HellaSwag dataset.

Validation so far:

- `python -m py_compile research/eval/synthetic_association_eval.py research/tests/test_synthetic_association_eval.py`
- `python -m pytest research/tests/test_synthetic_association_eval.py -q`
- Result: `8 passed`.

Hardening/tuning notes from Codex:

- Fixed target leakage: training input is now `[noun, relation, PAD]`, with the
  answer supplied only as the supervised target.
- Fixed degenerate mappings at vocab=20: noun→verb and noun→adjective are now
  one-to-one offsets rather than modulo multipliers that can collapse.
- Added relation-only baseline: it stays near chance, so the probe requires noun
  identity, not just relation type.
- Current default calibration: active vocab `40`, train steps `20`, lr `2e-3`.
  Quick CPU sweep showed:
  - relation-only baseline near chance (`score_mean` about 0.04 at 20 steps);
  - noun+relation toy learner partially learns (`score_mean` about 0.57 at 20 steps);
  - reference GPT2/Mamba/RWKV models learn strongly at 20 steps under this toy probe.

Headline: **nano_blimp's mean score saturates** for every architecture

- `class_coherence_acc` (right vs wrong class continuation): 0.94-1.00
- `binding_fidelity_acc` (correct vs neighbor association): 0.94-1.00

**The discriminating sub-test is `order_grammaticality_acc`** (model
prefers `[noun][query][verb]` over `[query][noun][verb]`):

| arch | order_grammaticality (v80/10) |
|---|---:|
| RWKV reference | 0.82 |
| local_attn_ssm_hybrid | 0.84 |
| **b0c38826 (top inducer)** | **0.62** |
| **Mamba reference** | **0.60** |

Mamba and latent_compress (state-tracking architectures) struggle with
positional ordering. RWKV and attention-class do better. **This is a real
architectural insight**: the order test exposes a known weakness of
recurrent state-tracking — bag-of-tokens behavior at sub-sequence positions.

**Recommendation**:
- Treat `order_grammaticality_acc` as the primary nano-BLiMP signal,
  not the mean. Score = order_grammaticality, not avg of three.
- Keep class_coherence and binding_fidelity as DIAGNOSTIC sub-fields
  (saturation = pass/fail floor; if a model can't even hit class_coherence
  it's broken).
- For codex's synthetic_association: stay at `vocab=80, steps=20` per
  codex's matrix; the score there discriminates GPT2 0.94 / Mamba 0.65 /
  RWKV 1.00 / baseline 0.48 — well-calibrated.
