## Performance & Hygiene Audit — codex (2026-03-31)

### Claimed Non-Duplicate Findings

1. **Predictor retraining inside screening hot path**
   - Files: `research/scientist/runner/execution_screening.py:1304-1351`, `research/scientist/intelligence/predictor.py:969-1038`, `research/scientist/intelligence/gnn_predictor.py:431-580`, `research/scientist/intelligence/interaction_model.py:191-255`
   - Problem: the prescreener trains the ensemble, topology predictor, interaction model, and calibration path during live screening instead of loading frozen artifacts.
   - Impact: inference path pays training cost, repeated SQLite scans, repeated JSON parsing, repeated NumPy fitting.
   - Fix direction: move training to offline artifact generation; screening only loads versioned predictor artifacts and cached op stats.

2. **Tropical routing and MoE are still Python-loop compute**
   - File: `research/mathspaces/tropical_routing.py:16-70, 114-145`
   - Problem: custom autograd and expert dispatch are controlled by Python loops and chunking.
   - Impact: this is real tensor compute stuck in Python orchestration; it will not scale.
   - Fix direction: replace with Triton/CUDA/C++ fused kernels or hard-disable from performance-sensitive execution until native path exists.

3. **Per-op runtime dispatch destroys native kernel gains**
   - File: `aria_designer/runtime/dispatch.py:19-220`
   - Problem: every call converts NumPy↔Torch and wraps small native ops individually.
   - Impact: allocator churn and conversion overhead erase much of the value of native kernels.
   - Fix direction: keep tensors in native representation end-to-end; dispatch whole subgraphs or modules, not isolated ops.

4. **Runtime fallbacks include fake math and placeholder identities**
   - Files: `aria_designer/components/math/sin/kernel_fallback.py:1-5`, `aria_designer/components/math/norm_last/kernel_fallback.py:1-5`, `aria_designer/components/math_space/low_rank_proj/kernel_fallback.py:1-22`
   - Problem: missing kernels silently degrade to identity behavior.
   - Impact: correctness corruption and false confidence during preview/eval/runtime.
   - Fix direction: either implement real fallback semantics or block execution when kernels are unavailable.

5. **Knowledge backfill endpoint is a live production stub**
   - File: `research/scientist/api_routes/knowledge_bp.py:35-43`
   - Problem: route exists, returns `501`, and tests explicitly tolerate that behavior.
   - Impact: dead surface area is being preserved instead of completed or removed.
   - Fix direction: implement migration target or remove/block the route until real code exists.

6. **Duplicate runtime registry implementations are drifting**
   - Files: `aria_designer/runtime/registry.py:1-144`, `aria_designer/runtime/compiler.py:190-279`
   - Problem: two registry loaders exist with overlapping responsibility; repo search shows the compiler-side one is the live path.
   - Impact: maintenance waste, divergence risk, dead-code camouflage.
   - Fix direction: delete or merge `runtime/registry.py`; keep one authoritative implementation.

7. **Profiling pipeline is not trustworthy**
   - Files: `research/runtime/native/bench/bench_e2e.py:53-79`, `conductor/profile_hotpaths.py:1-63`, `tasks/audit/profile_hotpaths.json`
   - Problem: benchmark harness currently fails due to broken import path, and recent perf artifact summary shows repeated budget failures with missing metrics.
