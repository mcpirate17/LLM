## Background you need

NanoBind is an S0 hard rejection gate, validated by independent matrices
from two prior sessions. Code is at `research/eval/nano_bind.py`; the
underlying corpus generator at `research/tools/nano_corpus_v0.py`. Joint
result: persistent slot-ending accuracy of EXACTLY 0.00 across a short
sweep ⇒ frequency-mode-collapse degenerate ⇒ reject.

**Hard rule (the only rejection criterion):**

```
is_no_go == True  ⟺  every checkpoint in the sweep returned exactly 0.00
                       slot-ending accuracy.
```

Anything else (0.01, 0.5, 1.0, mixed-zero-and-nonzero, error, timeout) is
NOT a rejection. NanoBind is a no-go gate only — passing means "eligible to
continue evaluation," never "approved" or "good."

The required call:

```python
from research.eval.nano_bind import nano_bind
result = nano_bind(graph_json_str, device="cuda", seed=0)
# result.is_no_go: bool   ← the only rejection signal
# result.scores: tuple of floats per checkpoint
# result.top5_tokens: per-cp/per-prompt audit trail
# result.sweep_metadata: dict
# result.status: 'ok' | 'compile_failed' | 'timeout' | 'error'
```

Default sweep is `corpus=280, vocab=10, strict_n_adj_per_noun=3,
checkpoints=(1000, 2000), held_out=cat/book/lamp/child/ship`. Do not
change unless asked.

   write. For each `result.is_no_go == True`:

   - `UPDATE leaderboard SET tier='screened_out' WHERE result_id=?`

      failure_details_json=? WHERE result_id=?`
     where `failure_details_json` is `json.dumps({"reason":
     "nano_bind_persistent_zero", "scores": result.scores,
     "metric_version": "nano_bind_v0",
     "checkpoints": list(result.sweep_metadata["checkpoints"])})`

   Do NOT modify rows where `result.is_no_go == False`. Do NOT modify
   rows where `result.status != 'ok'`. Errors get logged but do not
   change leaderboard state.

4. **Use a writer lock.** This project uses `aria-db` write coordination
   (see `feedback_aria_db_wal_hygiene.md` in the user's memory). Open
   one writer connection for the whole backfill. Never delete the
   `.writer-lock` file.

5. **Resume-safe.** If interrupted, re-running the script picks up where
   it left off (sidecar files are the resume marker). Verify before
   starting that the sidecar directory is fresh OR contains only
   completed results from a previous run.

6. **Report at the end:** total processed, total no-go, total pass,
