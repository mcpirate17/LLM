# Standalone Native Induction Probe

This directory contains an isolated induction-probe port that does not modify
or import any edited production files.

Files:

- `native_induction_probe.cpp`
  - C++/ATen generator for induction batches and pooled batch buffers.
- `fast_induction_probe.py`
  - Standalone fast probe runner using the native generator.
- `bench_fast_induction_probe.py`
  - Reproducible benchmark harness against the current Python induction probe.

Example runs:

```bash
python tasks/induction_native_probe/bench_fast_induction_probe.py
python tasks/induction_native_probe/bench_fast_induction_probe.py --steps 1000 --eval 100 --repeats 1
python tasks/induction_native_probe/bench_fast_induction_probe.py --steps 300 --repeats 3 --pool-size 64
```

Interpretation:

- `baseline_python`
  - Current `research.eval.induction_probe.induction_score`.
- `native_generator`
  - Same algorithm, but induction batches are generated by the standalone
    C++/ATen extension.
- `native_pool_N`
  - Native generator plus a reusable on-device batch pool. This is typically
    the fastest variant, but it changes the data schedule and can move AUC
    slightly compared with the baseline.

Recommended default:

- Use `native_generator` when you want speedup with the cleanest metric parity.
- Use `native_pool_N` only when you accept a small distribution shift in
  exchange for more speed.
