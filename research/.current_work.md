  it must either persist the normal post-S1 metrics (WikiText, HellaSwag, BLiMP,

### Phase 1 — Single source of truth for program_result kwargs
- New `research/scientist/runner/_helpers_metrics.py:program_result_kwargs_from_s1`
  is the canonical adapter from a `_micro_train` s1 dict to record_program_result
  kwargs. Bundles S1 scalars + screening + probes + routing + v9 trajectory +
  pruning + behavioral fingerprint columns + perf/kernel/starvation reports.
- Replaced the hand-rolled bundles in `runner/synthesis.py:_run_ablation_experiment`
  and in `tools/backfill_ablation_metrics.py:metric_patch_from_s1`. Future drift
  is impossible because both call ONE function.
- New guardrail in `notebook/program_writes.py:_enforce_s1_metric_completeness_for_ablation`
  refuses ANY `model_source='ablation'` write claiming `stage1_passed=True` while
  missing the 7 required post-S1 metrics (wikitext_perplexity, hellaswag_acc,
  blimp_overall_accuracy, induction_auc, binding_auc, binding_composite, ar_auc).
  Backfill replay paths bypass via `trust_label='ablation_metric_backfill_*'`.
- Constants exported: `S1_REQUIRED_POST_METRIC_COLUMNS`,
  `s1_post_metric_completeness()` (audit helper).

### Phase 2 — Collapse ablation entry points
- New `research/scientist/causal_attribution.py:run_ablation_suite` is the single
  canonical execution helper. Wraps history-mining + dedup + executor call +
  evidence summary + observation persistence + nb.flush_writes.
- `tools/champion_exhaustive_ablation.py:run_suite` and
  `runner/synthesis.py:_run_causal_ablation` are now thin wrappers that pass a
  campaign label to `run_ablation_suite`. Same code path.
- `causal_attribution.annotate_observations` is the public version of the
  per-fingerprint provenance overlay.

### Phase 3 — Diagnostics page redesigned
- New backend endpoints (each smoke-tested in test_ablation_api.py):
  - GET `/api/ablations/champions` — per-champion rollup with per-metric mean Δ.
  - GET `/api/ablations/components` — per-rule rollup, filterable by rule_type.
  - GET `/api/ablations/recommendations` — n≥4, metric_complete≥3 rules with
    multi-metric verdict tags (`use`/`avoid`/`mixed`/`inconclusive`).
  - GET `/api/ablations/children-for-rule` — drill-down with parent→child values
    on every probe.
  - GET `/api/ablations/construction-prior` and POST `/api/ablations/construction-prior/refresh`.
- Frontend `dashboard/src/components/AblationDiagnostics.js` rewritten as
  Banner + ConstructionPriorBar + ViewSwitcher (Recommendations / Champions /
  Components / All Rules) + DrillDrawer. Per-metric Δ columns
  (induction, binding, AR, BLiMP, HellaSwag, PPL%, loss) — never composite-only.
- Provenance shown as badges per child observation (executed/historical/
  reconstructed/loss-only).

### Phase 4 — Construction-prior loop wired in
- New `research/scientist/construction_priors.py`:
  - `compute_construction_prior(nb)` — reads ablation evidence + program_results,
    computes a multi-metric verdict score per rule with weights
    {induction:.20, binding:.20, ar:.10, blimp:.15, hellaswag:.10, ppl_pct:.15,
    loss:.10}, classifies use/avoid/mixed, builds op_weights/slot_motif_multipliers/
    slot_motif_denylist tuned for grammar consumption.
  - `record_construction_prior_snapshot(nb, prior, activate=True)` — versioned,
    immutable. Activating a new snapshot demotes the previous active one.
  - `get_active_construction_prior(nb)` and `list_construction_prior_snapshots`.
- New schema in `notebook/_shared.py`: `construction_prior_snapshots(version,
  created_at, is_active, payload_json, summary_json, notes)`.
- `runner/execution_screening.py` now overlays the active snapshot's adjustments
  on top of the legacy `causal_generation_adjustments`. Loud INFO log when applied.
- First snapshot created today: `v20260430-064458`, 77 rules (30 use / 1 avoid /
  46 mixed) covering 14 op_weights, 13 slot_motif_multipliers.
