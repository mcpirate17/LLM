## 1. Architecture Overview

```
┌──────────────────────────────────────────────────────────────────┐
│                        aria-designer/                            │
│                                                                  │
│  ui/               React + React Flow canvas                     │
│  api/              FastAPI — component registry, graph ops       │
│  runtime/          C/C++/Rust/Cython execution engine            │
│  components/       Packaged components (each = folder)           │
│  schemas/          Versioned JSON Schema contracts               │
│  tests/            Integration + contract tests                  │
│  .github/          CI/CD workflows                               │
│  Makefile          Build orchestration (native + python + ui)    │
└──────────────────────────────────────────────────────────────────┘
```

### Key Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Canvas library | React Flow (xyflow) | Already in use, best React ecosystem, virtualization for large graphs |
| Backend | FastAPI (Python) | Contract-first, async, Pydantic validation, matches research/ stack |
| Native kernels | C/Rust via cffi/PyO3 + Cython fallback | Priority: performance-critical ops in C/Rust, Python only for glue |
| Component format | Folder-per-component with manifest.yaml | Pipeline Pilot pattern — discoverable, testable, versionable |
| Graph contract | JSON Schema v1 (already exists) | Single source of truth, serializable, diffable |
| Execution model | Streaming DAG (Pipeline Pilot style) | Memory-efficient, supports large models, early error detection |
| Port typing | Typed ports with shape inference | Prevent invalid connections at wire-time, not execution-time |

### 2.2 manifest.yaml Schema

```yaml
id: "linear_proj"
version: "1.0.0"
name: "Linear Projection"
category: "linear_algebra"
tags: ["parameterized", "learnable", "core"]
status: "approved"                    # draft | approved | deprecated

description: "Learned dense linear projection (D_in -> D_out)"

# Port definitions — typed for connection validation
inputs:
  - name: "x"
    dtype: "tensor"
    shape: ["B", "S", "D_in"]        # Symbolic shape
    required: true

outputs:
  - name: "y"
    dtype: "tensor"
    shape: ["B", "S", "D_out"]

# Configurable parameters (rendered as property panel in UI)
params:
  out_dim:
    type: "integer"
    default: null                      # null = same as D_in
    description: "Output dimension. Defaults to input dimension."
    constraints: { min: 1, max: 65536 }
  bias:
    type: "boolean"
    default: false
  init_scheme:
    type: "enum"
    options: ["xavier", "kaiming", "orthogonal", "small"]
    default: "xavier"

# Implementation priority order
implementation:
  native: "kernel.c"                   # Preferred: C kernel compiled via cffi
  rust: null                           # Or: "kernel.rs" compiled via PyO3/maturin
  cython: null                         # Or: "kernel.pyx"
  python: "kernel_fallback.py"         # Fallback: pure Python/PyTorch

# Performance metadata
performance:
  has_params: true
  param_formula: "D_in * D_out"
  preserves_gradient: true
  numerically_risky: false
  flops_formula: "2 * B * S * D_in * D_out"

# Resource limits (enforced at runtime)
limits:
  max_memory_mb: 2048
  max_time_ms: 5000
  deterministic: true

# Compatibility constraints (maps to morphological_box constraints)
constraints:
  incompatible_with: []
  requires: []
```

Data/control example:

### 2.3 Component Handler Interface

Every component must implement this interface (in whatever language):

```c
// C interface (preferred for performance-critical ops)
// Compiled to shared library, loaded via cffi

typedef struct {
    float* data;
    int64_t* shape;
    int ndim;
    int dtype;  // 0=f32, 1=f16, 2=bf16
} TensorView;

typedef struct {
    const char* json_config;  // JSON-encoded params
} ComponentConfig;

// Required entry points
int component_validate(const ComponentConfig* config, char* error_buf, int buf_size);
int component_forward(const TensorView* inputs, int n_inputs,
                      TensorView* outputs, int n_outputs,
                      const ComponentConfig* config);
void component_cleanup(void);
```

```python
# Python fallback interface
from typing import Dict, Any, List
import torch

class ComponentHandler:
    """Base class for Python component implementations."""

    def validate_config(self, config: Dict[str, Any]) -> List[str]:
        """Return list of validation errors (empty = valid)."""
        return []

    def build(self, config: Dict[str, Any]) -> torch.nn.Module:
        """Build a PyTorch module from config."""
        raise NotImplementedError

    def forward(self, inputs: Dict[str, torch.Tensor],
                config: Dict[str, Any]) -> Dict[str, torch.Tensor]:
        """Execute the component. Keys match port names from manifest."""
```

### 3.2 Native Kernel Build System

```makefile
# runtime/Makefile
# Builds all native kernels into a single shared library

CC = gcc
CFLAGS = -O3 -march=native -fPIC -shared
RUST_FLAGS = --release

# C kernels
runtime/lib/libaria_kernels.so: runtime/src/*.c
	$(CC) $(CFLAGS) -o $@ $^

# Rust kernels (via maturin for PyO3)
runtime/lib/aria_rust_kernels.so:
	cd runtime/rust && maturin build --release

# Cython kernels
runtime/lib/aria_cython_kernels.so: runtime/cython/*.pyx
	cython -3 $^ && $(CC) $(CFLAGS) ...
```

### 3.3 Graph Validator (C implementation)

Core validation in C for speed (hundreds of graphs validated per second during search):

```
runtime/src/
├── graph_validator.c      # DAG validation, topological sort, cycle detection
├── shape_inference.c      # Shape propagation through typed ports
├── constraint_checker.c   # Morphological box constraint enforcement
├── graph_validator.h      # Public API
└── bindings.py            # cffi bindings to Python
```

### 3.4 Kernel Dispatch

```python
# runtime/dispatch.py
# Selects best available implementation for each component

class KernelDispatcher:
    def resolve(self, component_id: str, device: str) -> Callable:
        """Priority: native C/Rust > Cython > Python fallback."""
        manifest = self.registry.get(component_id)
        if manifest.implementation.native and device == "cpu":
            return self._load_c_kernel(manifest)
        if manifest.implementation.rust:
            return self._load_rust_kernel(manifest)
        if manifest.implementation.cython:
            return self._load_cython_kernel(manifest)
        return self._load_python_fallback(manifest)
```

### 4.3 Drag-and-Drop from Palette

```
1. User drags component from palette
2. onDragStart: set transferData = { componentId, category }

   a. Read component manifest from registry
   b. Create new node with default params
```

### 4.4 Connection Validation (at wire-time)

```
1. User drags edge from output port to input port
2. onConnect callback fires
3. Check: source port dtype matches target port dtype
4. Check: source port shape is compatible with target port shape
5. Check: no cycles would be introduced
6. Check: morphological_box constraints not violated
7. If invalid: reject connection, show tooltip with reason
8. If valid: add edge, trigger incremental shape inference
```

## 7. Aria Co-Design Protocol

### 7.1 Patch Operations

Aria proposes structured patches (never arbitrary code):

| Op | Description | Payload |
|----|-------------|---------|
| `add_node` | Insert new node | `{component_id, params, position, connect_to}` |
| `remove_node` | Remove node + reconnect | `{node_id, reconnect_strategy}` |
| `replace_node` | Swap component type | `{node_id, new_component_id, new_params}` |
| `rewire` | Change edge routing | `{edge_id, new_source, new_target}` |
| `mutate_param` | Adjust parameter | `{node_id, param_name, new_value}` |
| `create_block` | Group nodes into block | `{node_ids, block_name}` |

### 7.2 Human-in-the-Loop Approval

```
Aria proposes patch
      │
      ▼
[UI shows diff view: before/after side-by-side]
      │
      ├── User approves ──▶ patch applied, workflow versioned
      ├── User modifies ──▶ user edits patch, then applies
      └── User rejects  ──▶ logged with reason for learning
```

### 7.3 AI Agent Component Creation

AI agents (Claude, Copilot, Aria) can create new components:

```
1. Agent writes component folder (manifest.yaml + kernel + tests)
2. POST /api/v1/components  →  status = "draft"
3. Contract tests run automatically
4. If tests pass → eligible for approval
5. Human or auto-policy approves → status = "approved"
6. Component appears in palette
```

```yaml
# .github/workflows/ci.yml
name: Aria Designer CI
on: [push, pull_request]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: ruff check api/ runtime/
      - run: cd ui && npm ci && npm run lint

  test-python:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: "3.12" }
      - run: pip install -r api/requirements.txt -r runtime/requirements.txt
      - run: pytest tests/ -x --tb=short

  test-native:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: make -C runtime test    # Compile + run C/Rust kernel tests

  test-ui:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: cd ui && npm ci && npm test

  build-native:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: make -C runtime build   # Build shared libraries

  component-contracts:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: python -m pytest tests/test_component_contracts.py -x
```

## Notes for AI Agents

1. **Claim before working**: Update the task board above before starting.
2. **Native first**: Always write C/Rust kernel before Python fallback. Python is the last resort.
3. **Test everything**: Every component needs contract tests. Every API endpoint needs integration tests.
4. **Shape inference is critical**: Port shapes must propagate correctly — this is the #1 source of bugs.
5. **Don't break research/**: The adapter layer must translate cleanly. Run `pytest research/tests/ -x` after any bridge changes.
6. **manifest.yaml is the contract**: If the manifest says the component does X, the implementation must do exactly X.
